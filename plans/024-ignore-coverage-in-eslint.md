## Why this matters

Running `npm run lint` produces 3 warnings from generated coverage HTML/JS files (`coverage/block-navigation.js`, `coverage/prettify.js`, `coverage/sorter.js`). Each says "Unused eslint-disable directive" because these are third-party generated files with their own eslint annotations. These are not real issues — they force developers to visually filter noise when reviewing lint output.

## Current state

- `eslint.config.js` — ESLint flat config using `eslint/config` helper

Currently only `dist` is ignored:

```javascript
// eslint.config.js:9
globalIgnores(['dist']),
```

The `coverage/` directory (generated by `vitest --coverage`) is not excluded. Its JS files contain eslint-disable directives that trigger warnings.
